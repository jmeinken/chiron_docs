API - Transforming Cohort Def¶

Endpoints related to the transformation of the cohort def often have different request/response parameters based on the CohortDef processor associated with the concept.

transform_cohort_def_form¶

chiron.api.views.transform_cohort_def_form(request, form_name)¶

Some cohort_def transformation types need to collect user input using a form. This view returns data needed to render a form.

Parameters

  • request (django.http.request.HttpRequest) – The Djagno request

  • form_name (str) – The name of the form requested

Returns

The JSON data

Return type

json

CONCEPT FORM RESPONSE:

  • concept_id

  • concept_name

  • concept__plural_name

  • concept_description

  • category_hierarchy

  • processor_type: this is important since different processors return different form_data

  • form_data: a dict of data needed to render the form

The form_data returned from transform_cohort_def_form will depend on the processor type. Here are the built-in processor types:

form_data for CategoryFieldProcessor¶

CategoryFieldProcessor.get_concept_form_data(cd, existing_cd_entry=None)¶

  • form_errors: if post request, the list of strings describing errors validating the form

  • values: a list of dicts with category, count, and uniquePatientCount for each category

  • max_count: the count for the category with the greatest number of records

  • selected_values: if editing existing cd_entry, the values already selected

  • count_missing_values: number of records with null or missing values

  • count_subjects_with_missing_values: number of subjects who have records with null or missing values

  • exclude_selected: if editing existing cd_entry, True if exclude_selected is checked

form_data for TextFieldProcessor¶

TextFieldProcessor.get_concept_form_data(cd, existing_cd_entry={})¶

Returns a data dict of values needed to render the concept form/visualization. Typically, this will merge together data from self._get_preprocessed_data() with any data from the provided existing_cd_entry

Parameters

  • cd (list of dicts) – the cohort definition (typically visualizations will be specific for provided cohort def)

  • existing_cd_entry (dict) – cohort def entry if editing existing

  • query_type (string) – some concepts might have multiple forms for multiple types of queries

Returns

data needed to render form

Return type

dict
