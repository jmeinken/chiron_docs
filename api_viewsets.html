Concepts¶

/concepts/ (GET)¶

ConceptViewSet.list(request, *args, **kwargs)¶

Retrieve a list of concepts

Parameters

  • collection (string, optional) – permanent_id for a collection to filter concepts

  • category (int, optional) – id for a category to filter concepts

  • search (search Concepts by name, name plural, and description) – search Concepts by name, name plural, and description

  • show_unpublished (boolean, default=False) – set this to “true” to return concepts that are not published

  • concept_type (string, cohort|table, default=None) – filter based on the concept type

Returns

serialized list of concepts

Return type

list

/concepts/[permanent_id]/ (GET, PUT)¶

ConceptViewSet.retrieve(request, permanent_id=None)¶
Parameters

  • include_statistics (boolean, default=False) – include statistical analysis of concept

  • cohort_def (list, optional) – the current cohort definition for calculating statistics, if None then the active cohort definition will be used

  • show_all_data (boolean, default=False) – When calculating statistics, ignore the cohort_def and return stats about data for all subjects

  • include_cohort_def_options (boolean, default=False) – include options for creating a cohort def entry

  • include_table_def_options (boolean, default=False) – include options for creating a table def entry

  • cohort_def_entry_id (int, optional) – if editing an existing cohort def entry, the entry_id to use whengenerating cohort def options

  • table_def_entry_id (int, optional) – if editing an existing table def entry, the entry_id to use when generating table def options

  • callback (boolean, default=False) – run callback method for concept

Returns

A serialized concept; plus optionally statistics, cohort_def_options, and callback_data

ConceptViewSet.update(request, permanent_id=None)¶

Does the same thing as retrieve() but allows a PUT request for long query parameters.

ConceptCategories¶

/concept_categories/ (GET)¶

ConceptCategoryViewSet.list(request, *args, **kwargs)¶

Returns all top-level categories and concepts (i.e. categories that don’t have parents and concepts that don’t have a category). Results are filtered based on user permissions.

Parameters

concept_type (string, cohort|table, default=None) – filter based on the concept type

RESPONSE:

Category_id

“top”

Categories

(list) all categories without a parent

Concepts

(list) all concepts without a category

/concept_categories/[category_id]/ (GET)¶

ConceptCategoryViewSet.retrieve(request, pk, *args, **kwargs)¶

Parameters

RESPONSE:

the provided category ID

Concepts

(list) all concepts belonging to this category

QueryTools¶

/query_tools/count/ (GET, POST)¶

QueryToolsViewSet.count(request, *args, **kwargs)¶

Get the count of subjects in a cohort.

The POST method is provided to allow passing of long query parameters. It works exactly the same as the GET method. :param cohort_def: The cohort def to use. If not provided, the active cohort_def or an

empty cohort def will be used.

RESPONSE

Count

(integer) The number of subjects in the cohort

Cohort_def

The cohort def that was used.

/query_tools/preview/ (GET, POST)¶

QueryToolsViewSet.preview(request, *args, **kwargs)¶

Get a paginated dataset.

The POST method is provided to allow passing of long query parameters. It works exactly the same as the GET method.

Parameters

  • cohort_def (list, optional) – The cohort def to use. If not provided, the active cohort_def will be used if available or an empty cohort def.

  • table_def (list, optional) – The table_def to use. If not provided, the active table_def will be used if available or an empty table_def.

  • records_per_page (int, default=10) – How many records per page when paginating

  • page (int, default=1) – The page number to fetch

RESPONSE IF SUCCESS:

Warnings

(array of strings) Warning messages to display to the user

Record_count

(int)

Subject_count

(int)

Extended_table_def

(list)

Paginator

(dict of integers) includes first_index, last_index, current_page, previous_page, next_page, last_page, total_records

Data

(2D array of strings) The records returned for this page; if a value is also associated with a link, a concept return a dict with “link” and “value” instead of a string

RESPONSE IF FAIL:

Warnings

(array of strings) Warning messages to display to the user

Errors

(array of strings) Error messages that explain why the query failed

Preview_failed

True

Extended_table_def

(list)

ReportTools¶

/report_tools/[report_id]/share/ (POST)¶

ReportToolsViewSet.share(request, pk, *args, **kwargs)¶

Pass an array of all ChironUser IDs to share with. Any preexisting sharing will be erased.

Note that reports set to “public” are automatically shared with everyone regardless of what’s set with sharing.

/report_tools/[report_id]/load_as_active (POST)¶

ReportToolsViewSet.load_as_active(request, pk, *args, **kwargs)¶

Will set this report’s cohort def and table def as the active snapshot in the CohortDefSnapshot and TableDefSnapshot models. It will also erase the snapshot history in both tables.
